# ‚è≥ JavaScript Asynchronous Programming

---

## üìñ Definition

**Asynchronous programming** in JavaScript allows code to **run without blocking** the execution of other tasks.
It‚Äôs essential for handling tasks like:

* Fetching data from APIs
* Reading/writing files (Node.js)
* Timers and animations
* User interactions

JavaScript uses an **event loop** to manage asynchronous tasks.

---

## üîç Key Concepts

| Concept          | Description                                                              |
| ---------------- | ------------------------------------------------------------------------ |
| **Synchronous**  | Code runs line by line; one task must finish before the next starts.     |
| **Asynchronous** | Code does not block execution; tasks can run in the background.          |
| **Callbacks**    | Functions passed as arguments, executed after an async task completes.   |
| **Promises**     | Objects representing future values (resolved or rejected).               |
| **Async/Await**  | Modern syntax for handling promises, making async code look synchronous. |

## üß± 1. Synchronous Example

```javascript

console.log("Task 2");
console.log("Task 3");
```

‚úÖ Output order:

```

```

## üß± 2. Asynchronous with `setTimeout`

```javascript
console.log("Task 1");

setTimeout(() => {
  console.log("Task 2 (async)");
}, 2000);

console.log("Task 3");
```

‚úÖ Output order:

```
Task 1
Task 3
Task 2 (async)
```

üí° `Task 2` runs later due to the **event loop**.

## üß± 3. Callback Functions

```javascript
function fetchData(callback) {
  console.log("Fetching data...");
  setTimeout(() => {
    callback("Data received!");
  }, 2000);
}

fetchData((result) => {
  console.log(result);
});
```

‚úÖ Problem: Callbacks can lead to **callback hell** (nested functions).

---

## üß± 4. Promises

```javascript
let promise = new Promise((resolve, reject) => {
  let success = true;

  if (success) {
    resolve("Data loaded successfully!");
  } else {
    reject("Error loading data!");
  }
});

promise
  .then((result) => console.log(result))   // success handler
  .catch((error) => console.log(error))   // error handler
  .finally(() => console.log("Operation complete"));
```

---

## üß± 5. Chaining Promises

```javascript
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => response.json())
  .then(data => {
    console.log("Post Title:", data.title);
    return fetch("https://jsonplaceholder.typicode.com/users/1");
  })
  .then(response => response.json())
  .then(user => console.log("Author:", user.name))
  .catch(error => console.log("Error:", error));
```

---

## üß± 6. Async/Await

```javascript
async function getData() {
  try {
    let response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
    let data = await response.json();
    console.log("Post:", data.title);
  } catch (error) {
    console.log("Error:", error.message);
  }
}

getData();
```

üí° `async/await` makes asynchronous code **look synchronous**.

---

## üß± 7. Parallel Async with `Promise.all`

```javascript
async function fetchAll() {
  try {
    let [post, user] = await Promise.all([
      fetch("https://jsonplaceholder.typicode.com/posts/1").then(res => res.json()),
      fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json())
    ]);

    console.log("Post:", post.title);
    console.log("User:", user.name);
  } catch (error) {
    console.log("Error:", error.message);
  }
}

fetchAll();
```

## üåç Real-World Example ‚Äì API Call

```javascript
async function loginUser(username, password) {
  try {
    let response = await fetch("https://api.example.com/login", {
      method: "POST",
      body: JSON.stringify({ username, password }),
      headers: { "Content-Type": "application/json" }
    });

    if (!response.ok) throw new Error("Login failed!");

    let data = await response.json();
    console.log("Welcome,", data.user);
  } catch (error) {
    console.log("Error:", error.message);
  }
}

loginUser("ramya", "mypassword");
```

## ‚úÖ Summary Table

| Feature         | Description                                              |
| --------------- | -------------------------------------------------------- |
| **Callbacks**   | Oldest async method, can get messy (callback hell).      |
| **Promises**    | Cleaner async handling, with `.then()` and `.catch()`.   |
| **Async/Await** | Modern approach, makes async look synchronous.           |
| **Event Loop**  | JS mechanism that handles async tasks in the background. |

---

## üß† Best Practices

1. ‚úÖ Use **async/await** instead of nested callbacks.
2. ‚úÖ Handle errors with `try...catch` or `.catch()`.
3. ‚úÖ Use **`Promise.all`** for parallel tasks.
4. ‚ùå Avoid **callback hell** by breaking code into smaller functions.
5. ‚úÖ Always validate async API responses before using them.
